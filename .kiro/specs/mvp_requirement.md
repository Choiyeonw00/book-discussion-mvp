# MVP 기능 확정

<aside>
💡

**MVP는 뭐고, 왜 정하는 걸까요?**

- MVP는 ‘최소 기능 구현 프로덕트’입니다. 우리가 발견한 문제를 해결할 가설이 타당한지 확인하는 데 필요한 최소한의 기능을 구현합니다.
- 우리가 무엇을 검증할 것인지 정하기 위함입니다.
    - “기록이 토론 참여로 이어지는가?”
- 개발 범위를 통제하기 위함입니다.
    - 각 이터레이션마다 정해진 최소 기능 구현을 통해 일정을 조율하고, 프로젝트 규모를 점진적으로 키워나갈 수 있습니다.
</aside>

<aside>
💡

### MVP

사용자가 독서 중 남긴 메모를 기반으로, 자연스럽게 토론에 참여할 수 있는 최소 기능을 구현한다.

</aside>

### MVP 기능 리스트

**핵심 기능**

- 로그인/회원가입 - 간단하게
- 책 기반 모임 생성/참여
- 메모 작성
- 메모 공유
- 메모 기반 토론

**차별화 기능**

- 메모 기반 토론 주제 추천

# User flow

<aside>
💡

**User flow는 뭐고, 왜 정하는 걸까요?**

- user flow: 사용자가 서비스를 이용하면서 목표를 달성하기까지의 행동 흐름
    - 모임에 들어간다 → 책을 읽으며 메모를 남긴다 → 메모를 공유한다 → 토론에 참여한다
- User flow를 그리면 우리가 무엇을 만들지 명확해집니다.
    - 기능 목록만 보는 것보다 확실하고 빠르게 와닿습니다
- 불필요한 기능을 제거할 수 있습니다.
    - User flow에 들어가지 않는 기능은 당장 불필요한 기능이므로 MVP에 들어가지 않습니다.
- 팀원 간 이해를 맞출 수 있습니다.
    - User flow라는 거대한 그림을 백, 프론트, PM이 함께 보면서 각자 다른 서비스를 만드는 상황을 방지합니다.
</aside>

### 간략한 User flow, MVP 기준

모임 입장 → 책 선택 → 메모 작성 → 메모 공유 → 토론 참여

- 이 흐름으로 서비스가 완결되며, 중간에 막히는 구간이 없고, 불필요한 단계가 없음
- 나올 수 있는 질문 (왜냐면 내가 궁금해서 공부해봄)
    
    **그럼 User flow에 있는 것만 개발하고, 나머지는 개발 안하나요?**
    
    - user flow에 없어도 ‘모임 생성’같은 기능은 개발해야 합니다.
    - 이 유저 플로우는 참여자 기준으로 작성된거지, 방장 기준으로 작성된 게 아니기 때문입니다.
    
    **해당 기능을 동작시키기 위한 백엔드, 서버/인프라 구축도 해야 하나요?**
    
    - 넵!! 실제 서비스를 제작중인 것이기 때문에 뒤에 시스템도 함께 구축합니다!!
    - 즉, 프론트, 백, DB설계, 배포 모두 진행해야해서 보기보다 할 일이 많습니다!

# 페이지 구조 정리

<aside>
💡

**페이지 구조를 정리의 목적!**

- 어떤 페이지가 필요한지
- 각 페이지에서 어떤 작업이 필요한지
- 프론트/백/기획 task가 어떤 게 있는지

놓치지 않기 위해 가볍게 정리해보는 시간이라고 생각하면 됩니다.

</aside>

### 1) 홈 화면 / 모임 리스트 페이지

**목적**

- 사용자가 참여할 독서 모임을 찾는다.

**핵심 요소**

- 진행 중인 모임 목록
- 책 제목 기준 검색
- 모임 카드(모임 소개?)
    - 책 제목
    - 책의 줄거리
    - 모임 소개
    - 독서 기간
    - 토론 날짜
    - 참여 인원

**주요 액션**

- 모임 검색
- 모임 상세 페이지로 이동
- 모임 생성 페이지로 이동

### 2) 모임 생성 페이지

**목적**

- 방장이 책 기준으로 독서 모임을 생성한다.

**핵심 요소**

- 책 제목 입력
- 줄거리 간략 소개(입력/더 쉬운 방법이 있다면…?)
- 모임명 입력
- 방 소개
- 모집 인원
- 독서 기간
- 토론 날짜

**주요 액션**

- 모임 생성
- 취소(뒤로가기)

### 3) 모임 상세 페이지

**목적**

- 모임 정보를 보고 참여하거나, 참여 후 활동 진입점(메모 등) 제공.

**핵심 요소**

- 책 정보
- 모임 정보
- 참여자 목록 + 참여자들의 독서 진행률
- 독서 기간 / 토론 일정
- 내 진행 상태 요약
- 최근 메모 / 최근 토론 주제(쓰레드)

**주요 액션**

- 모임 참여하기
- 메모 작성 페이지로 이동 (하루 독서 기록)
- 토론 페이지로 이동 (토론 진행 시 활성화)

### 4) 메모 작성 / 메모 리스트 페이지

**목적**

- 독서 중 생각을 남기고, 공개 여부를 설정하며, 다른 사람의 메모를 본다.

**핵심 요소**

- 읽은 범위 입력
- 메모 내용 입력
- 공개/비공개 설정
- 내 메모 목록
- 공개된 다른 사람 메모 목록 (내 독서 진행률에 따라 읽기 가능 여부 달라짐)

**주요 액션**

- 메모 작성
- 메모 수정/삭제
- 공개 여부 변경
- 특정 메모를 토론 주제로 연결

### 5) 토론 페이지

**목적**

- 메모나 주제를 기반으로 텍스트 토론을 진행한다.

**핵심 요소**

- 토론 주제 목록
- 스레드 별 댓글
- 메모 기반 추천 주제
- 내 작성 내역

**주요 액션**

- 주제 생성
- 스레드 입장
- 의견 작성
- 댓글 작성

### 6) 마이 페이지/활동 기록 페이지

**목적**

- 내가 참여한 모임과 메모, 토론 기록을 확인한다.

**핵심 요소**

- 참여 중인 모임
- 작성한 메모
- 참여한 토론
- 책별 활동 내역

**주요 액션**

- 특정 모임으로 이동
- 내 기록 확인
