# 요구사항 문서

## 소개

독서 토론 웹사이트의 MVP 기능을 정의한다. 사용자가 독서 중 남긴 메모를 기반으로 자연스럽게 토론에 참여할 수 있는 최소 기능을 구현하는 것이 목표이다. 핵심 가설은 "기록이 토론 참여로 이어지는가?"를 검증하는 것이다.

## 용어 사전

- **시스템**: 독서 토론 웹 애플리케이션 전체
- **인증_모듈**: 사용자 로그인 및 회원가입을 처리하는 모듈
- **모임**: 특정 책을 기반으로 생성된 독서 토론 그룹
- **방장**: 모임을 생성한 사용자
- **참여자**: 모임에 참여한 사용자
- **메모**: 사용자가 독서 중 작성한 텍스트 기록 (읽은 범위, 내용, 공개 여부 포함)
- **토론_주제**: 메모 또는 사용자 입력을 기반으로 생성된 토론 스레드
- **스레드**: 하나의 토론 주제에 달린 의견과 댓글의 묶음
- **독서_진행률**: 사용자가 해당 책에서 읽은 범위를 기반으로 산출한 진행 상태
- **추천_엔진**: 공개된 메모를 분석하여 토론 주제를 추천하는 기능 모듈
- **모임_카드**: 홈 화면에서 모임 정보를 요약하여 보여주는 UI 요소
- **책_검색_API**: 외부 도서 검색 API (카카오/네이버)를 통해 책 제목, 저자, 표지, 줄거리 등의 정보를 조회하는 모듈

## 요구사항

### 요구사항 1: 회원가입

**사용자 스토리:** 사용자로서, 계정을 생성하여 독서 토론 서비스에 참여하고 싶다.

#### 인수 조건

1. WHEN 사용자가 회원가입 양식을 제출하면, THE 인증_모듈 SHALL 이메일 형식과 비밀번호 유효성을 검증한 후 계정을 생성한다.
2. IF 이미 등록된 이메일로 회원가입을 시도하면, THEN THE 인증_모듈 SHALL "이미 사용 중인 이메일입니다" 오류 메시지를 표시한다.
3. IF 비밀번호가 8자 미만이면, THEN THE 인증_모듈 SHALL "비밀번호는 8자 이상이어야 합니다" 오류 메시지를 표시한다.

### 요구사항 2: 로그인

**사용자 스토리:** 사용자로서, 계정에 로그인하여 서비스 기능을 이용하고 싶다.

#### 인수 조건

1. WHEN 사용자가 올바른 이메일과 비밀번호를 입력하면, THE 인증_모듈 SHALL 인증 토큰을 발급하고 홈 화면으로 이동시킨다.
2. IF 이메일 또는 비밀번호가 일치하지 않으면, THEN THE 인증_모듈 SHALL "이메일 또는 비밀번호가 올바르지 않습니다" 오류 메시지를 표시한다.
3. WHILE 사용자가 로그인 상태이면, THE 시스템 SHALL 인증이 필요한 모든 페이지에 접근을 허용한다.
4. WHEN 인증 토큰이 만료되면, THE 인증_모듈 SHALL 사용자를 로그인 페이지로 리다이렉트한다.

### 요구사항 3: 모임 생성

**사용자 스토리:** 방장으로서, 책 기반 독서 모임을 생성하여 다른 사용자들과 함께 독서하고 싶다.

#### 인수 조건

1. WHEN 방장이 모임 생성 페이지에서 책 제목을 입력하면, THE 시스템 SHALL 책_검색_API를 호출하여 검색 결과(책 제목, 저자, 표지 이미지, 줄거리)를 목록으로 표시한다.
2. WHEN 방장이 검색 결과에서 책을 선택하면, THE 시스템 SHALL 해당 책의 제목, 저자, 표지 이미지, 줄거리를 모임 생성 양식에 자동으로 입력한다.
3. IF 책_검색_API 호출에 실패하거나 검색 결과가 없으면, THEN THE 시스템 SHALL 방장이 책 제목과 줄거리를 직접 입력할 수 있도록 수동 입력 모드를 제공한다.
4. WHEN 방장이 모임 생성 양식을 제출하면, THE 시스템 SHALL 책 정보, 모임명, 방 소개, 모집 인원, 독서 기간, 토론 날짜를 저장하고 모임을 생성한다.
5. IF 필수 입력 항목(책 제목, 모임명, 모집 인원, 독서 기간, 토론 날짜)이 누락되면, THEN THE 시스템 SHALL 누락된 항목을 명시하는 오류 메시지를 표시한다.
6. WHEN 모임이 생성되면, THE 시스템 SHALL 모임 생성자를 해당 모임의 방장이자 첫 번째 참여자로 등록한다.

### 요구사항 4: 모임 목록 조회 및 검색

**사용자 스토리:** 사용자로서, 진행 중인 독서 모임을 탐색하고 원하는 모임을 찾고 싶다.

#### 인수 조건

1. WHEN 사용자가 홈 화면에 접근하면, THE 시스템 SHALL 진행 중인 모임 목록을 모임_카드 형태로 표시한다.
2. THE 모임_카드 SHALL 책 제목, 줄거리, 모임 소개, 독서 기간, 토론 날짜, 참여 인원 수를 포함한다.
3. WHEN 사용자가 책 제목으로 검색하면, THE 시스템 SHALL 검색어를 포함하는 모임 목록을 필터링하여 표시한다.
4. IF 검색 결과가 없으면, THEN THE 시스템 SHALL "검색 결과가 없습니다" 메시지를 표시한다.

### 요구사항 5: 모임 참여

**사용자 스토리:** 사용자로서, 관심 있는 독서 모임에 참여하여 함께 독서하고 싶다.

#### 인수 조건

1. WHEN 사용자가 모임 상세 페이지에서 "참여하기" 버튼을 클릭하면, THE 시스템 SHALL 해당 사용자를 모임 참여자로 등록한다.
2. IF 모임의 참여 인원이 모집 인원에 도달하면, THEN THE 시스템 SHALL "모집 인원이 마감되었습니다" 메시지를 표시하고 참여를 차단한다.
3. IF 사용자가 이미 해당 모임에 참여 중이면, THEN THE 시스템 SHALL "이미 참여 중인 모임입니다" 메시지를 표시한다.

### 요구사항 6: 모임 상세 조회

**사용자 스토리:** 참여자로서, 모임의 상세 정보와 활동 현황을 확인하고 싶다.

#### 인수 조건

1. WHEN 사용자가 모임 상세 페이지에 접근하면, THE 시스템 SHALL 책 정보, 모임 정보, 참여자 목록, 독서 기간, 토론 일정을 표시한다.
2. WHILE 사용자가 모임에 참여 중이면, THE 시스템 SHALL 참여자별 독서_진행률을 표시한다.
3. WHILE 사용자가 모임에 참여 중이면, THE 시스템 SHALL 해당 사용자의 최근 메모와 최근 토론_주제를 요약하여 표시한다.
4. WHILE 사용자가 모임에 참여 중이면, THE 시스템 SHALL 메모 작성 페이지와 토론 페이지로의 이동 링크를 제공한다.

### 요구사항 7: 메모 작성

**사용자 스토리:** 참여자로서, 독서 중 떠오른 생각을 메모로 기록하고 싶다.

#### 인수 조건

1. WHEN 참여자가 메모 작성 양식을 제출하면, THE 시스템 SHALL 읽은 범위, 메모 내용, 공개 여부를 저장한다.
2. IF 메모 내용이 비어 있으면, THEN THE 시스템 SHALL "메모 내용을 입력해주세요" 오류 메시지를 표시한다.
3. THE 시스템 SHALL 메모 작성 시 공개/비공개 설정의 기본값을 비공개로 지정한다.

### 요구사항 8: 메모 관리

**사용자 스토리:** 참여자로서, 작성한 메모를 수정하거나 삭제하고, 공개 여부를 변경하고 싶다.

#### 인수 조건

1. WHEN 참여자가 본인의 메모를 수정하면, THE 시스템 SHALL 변경된 내용을 저장한다.
2. WHEN 참여자가 본인의 메모를 삭제하면, THE 시스템 SHALL 해당 메모를 삭제하고 메모 목록에서 제거한다.
3. WHEN 참여자가 메모의 공개 여부를 변경하면, THE 시스템 SHALL 변경된 공개 상태를 즉시 반영한다.
4. IF 다른 참여자의 메모를 수정 또는 삭제하려고 시도하면, THEN THE 시스템 SHALL 해당 요청을 거부하고 "본인의 메모만 수정/삭제할 수 있습니다" 메시지를 표시한다.

### 요구사항 9: 메모 공유 및 열람

**사용자 스토리:** 참여자로서, 다른 참여자의 공개 메모를 읽고 독서 경험을 공유하고 싶다.

#### 인수 조건

1. WHEN 참여자가 메모 리스트 페이지에 접근하면, THE 시스템 SHALL 본인의 전체 메모 목록과 다른 참여자의 공개 메모 목록을 구분하여 표시한다.
2. WHILE 참여자의 독서_진행률이 해당 메모의 읽은 범위보다 낮으면, THE 시스템 SHALL 해당 공개 메모의 내용을 숨기고 "해당 범위까지 읽은 후 열람할 수 있습니다" 메시지를 표시한다.
3. WHILE 참여자의 독서_진행률이 해당 메모의 읽은 범위 이상이면, THE 시스템 SHALL 해당 공개 메모의 전체 내용을 열람 가능하게 한다.

### 요구사항 10: 토론 주제 생성

**사용자 스토리:** 참여자로서, 메모를 기반으로 토론 주제를 생성하여 다른 참여자와 의견을 나누고 싶다.

#### 인수 조건

1. WHEN 참여자가 토론 주제 생성 양식을 제출하면, THE 시스템 SHALL 주제 제목과 내용을 저장하고 새로운 스레드를 생성한다.
2. WHEN 참여자가 특정 메모를 토론 주제로 연결하면, THE 시스템 SHALL 해당 메모를 토론_주제의 원본 메모로 참조 연결한다.
3. IF 토론 주제 제목이 비어 있으면, THEN THE 시스템 SHALL "토론 주제를 입력해주세요" 오류 메시지를 표시한다.

### 요구사항 11: 토론 참여

**사용자 스토리:** 참여자로서, 토론 스레드에서 의견과 댓글을 작성하여 토론에 참여하고 싶다.

#### 인수 조건

1. WHEN 참여자가 토론 페이지에 접근하면, THE 시스템 SHALL 해당 모임의 토론_주제 목록을 표시한다.
2. WHEN 참여자가 스레드에 의견을 작성하면, THE 시스템 SHALL 해당 의견을 스레드에 저장하고 표시한다.
3. WHEN 참여자가 의견에 댓글을 작성하면, THE 시스템 SHALL 해당 댓글을 의견 하위에 저장하고 표시한다.
4. WHILE 참여자가 토론 페이지에 있으면, THE 시스템 SHALL 해당 참여자의 작성 내역을 별도로 확인할 수 있는 필터를 제공한다.

### 요구사항 12: 메모 기반 토론 주제 추천

**사용자 스토리:** 참여자로서, 공유된 메모를 기반으로 토론 주제를 추천받아 토론을 활성화하고 싶다.

#### 인수 조건

1. WHEN 모임 내 공개 메모가 2개 이상 존재하면, THE 추천_엔진 SHALL 공개 메모의 내용을 분석하여 토론 주제 후보를 생성한다.
2. THE 추천_엔진 SHALL 추천된 토론 주제를 토론 페이지의 "추천 주제" 영역에 표시한다.
3. WHEN 참여자가 추천 주제를 선택하면, THE 시스템 SHALL 해당 추천 주제를 기반으로 새로운 토론_주제 스레드를 자동 생성한다.

### 요구사항 13: 마이페이지

**사용자 스토리:** 사용자로서, 참여한 모임과 작성한 메모, 토론 기록을 한곳에서 확인하고 싶다.

#### 인수 조건

1. WHEN 사용자가 마이페이지에 접근하면, THE 시스템 SHALL 참여 중인 모임 목록, 작성한 메모 목록, 참여한 토론 목록을 표시한다.
2. WHEN 사용자가 마이페이지에서 특정 모임을 선택하면, THE 시스템 SHALL 해당 모임의 상세 페이지로 이동시킨다.
3. WHEN 사용자가 마이페이지에서 특정 메모를 선택하면, THE 시스템 SHALL 해당 메모가 속한 모임의 메모 리스트 페이지로 이동시킨다.
4. WHEN 사용자가 마이페이지에서 특정 토론을 선택하면, THE 시스템 SHALL 해당 토론_주제의 스레드 페이지로 이동시킨다.
